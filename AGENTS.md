# AGENTS.md

## Project Overview
Static homepage for [darksidenoodles.com](https://darksidenoodles.com). This is a simple, single-page marketing site — not an application.

## Stack
- **Astro 6** — static site generator (no SSR, fully static output)
- **Tailwind CSS v4** — styling via `@tailwindcss/vite` plugin (no `tailwind.config` file; v4 uses CSS-based config)
- **TypeScript** — strict mode (`astro/tsconfigs/strict`)
- **Firebase Hosting** — deployment target, config in `firebase.json`, serves from `dist/`

## Project Structure
```
src/
  layouts/Layout.astro   — base HTML layout (head, meta, shared structure)
  pages/index.astro      — homepage (only page)
  styles/global.css      — global styles
  styles/sr-only.css     — screen-reader utility
public/                  — static assets (favicons, images, robots.txt)
```

## Key Files
- `astro.config.mjs` — Astro config; sets `site` URL, enables `@astrojs/sitemap`
- `firebase.json` — Firebase Hosting config with caching headers and security headers
- `public/robots.txt` — allows all crawlers, references sitemap

## Build & Dev
- `npm run dev` — local dev server on port 4321
- `npm run build` — outputs static site to `dist/`
- `firebase deploy` — deploys `dist/` to Firebase Hosting

## Conventions
- This is a static site with no backend, no API routes, and no client-side JavaScript frameworks
- Styles use Tailwind CSS v4 utility classes directly in `.astro` templates
- All images and static assets go in `public/`
- SEO metadata (Open Graph, structured data, sitemap) is managed in `Layout.astro`
- The sitemap is auto-generated by `@astrojs/sitemap` at build time

## Things to Avoid
- Do not add SSR or server endpoints — this is a purely static site
- Do not add a `tailwind.config.js/ts` — Tailwind v4 uses CSS-based configuration
- Do not manually edit `dist/` — it is a build artifact
